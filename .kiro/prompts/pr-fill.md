# GitHub Pull Request Creation


** NOTE ** use command `mygh` for all `gh` commands. `mygh` contains my personal GITHUB_TOKEN. It is simple wrapper around gh


Use your available resources and knowledge on pull request creation guidelines for GitHub.

Always add near the top of the pull request body that this software is prerelease.
As such, CHANGELOG.md and our documentation maybe out of sync. This is expected.
Also if there are breaking API or CLI changes, that is acceptable.


## Labels

Before submission, ask the user if they want to add `jobtaker` label.
This label is for automated AI code review


Proceed with gathering the information required to create a pull request
